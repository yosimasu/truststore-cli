### File Locations and Project Structure
[Source: docs/architecture/08-8-source-tree.md]
- Entry point: `cmd/truststore/main.go` - Main entry point and Cobra command setup
- CLI commands: `internal/app/list.go` - CLI command handlers
- Go module: `go.mod` at project root
- Version file: `.tool-versions` at project root for asdf
- Build automation: `Makefile` at project root
- Dist folder: `dist/` for build outputs (should be gitignored)

### Coding Standards
[Source: docs/architecture/11-11-coding-standards.md]
- **Error Handling**: Never discard errors with `_`, use `fmt.Errorf` with `%w` for wrapping
- **Naming**: Follow Go community conventions (camelCase for locals, PascalCase for exports)
- **No CGo**: Must remain pure Go for maximum portability
- **Linting**: All code must pass `golangci-lint` before commit via `make lint`

### Testing Requirements
[Source: docs/architecture/12-12-test-strategy-and-standards.md]
- **Framework**: Use Go's built-in `testing` package
- **File Convention**: `*_test.go` files co-located with source code
- **Coverage Goal**: Aim for >80% unit test coverage
- **Test Organization**: Unit tests in same package as code being tested
- **Mocking**: Use Go interfaces for dependency mocking (no external libraries)

### Technical Constraints
- Must be pure Go (no CGo)
- Single binary output for cross-platform distribution
- Standard Go project layout as defined in source tree
