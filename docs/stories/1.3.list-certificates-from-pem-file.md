# Story 1.3: List Certificates from a PEM file

## Status
Done

## Story
**As a** developer,
**I want** to run `truststore list my_certs.pem`,
**so that** I can verify the contents of a PEM-formatted certificate file.

## Acceptance Criteria
1. The `list` command can distinguish between a domain and a local file path argument.
2. The CLI can read and parse one or more certificates from a specified PEM file.
3. Certificate details are printed to the console in the same format used for remote lookups.
4. File-not-found and parsing errors are handled gracefully.

### Data Models
[Source: docs/architecture/04-4-data-models.md]
- **Certificate**: Use Go's standard `crypto/x509.Certificate` struct which includes:
  - `Subject` and `Issuer`  
  - `SerialNumber`
  - `NotBefore`, `NotAfter` (Validity Period)
  - `PublicKey` and `SignatureAlgorithm`
  - `Extensions` (including AIA for potential future use)
- **Truststore Interface**: Must implement common interface with methods like `ReadCertificates()` to provide generic certificate container abstraction regardless of file format
- **Relationships**: A `Certificate` can be an issuer for another `Certificate`, forming a chain relationship

### Architecture Patterns
[Source: docs/architecture/02-2-high-level-architecture.md]
- **Strategy Pattern**: Each truststore format (PEM, JKS, PKCS12) will be handled by a specific "strategy" (a struct that implements a common `Truststore` interface with methods like `Read`, `Add`, `Remove`)
- **Dependency Injection**: Core services will be initialized once and passed into commands that need them for improved testability and clear dependency management
- **Command Pattern**: Cobra library inherently uses command pattern to encapsulate all information needed to perform actions
